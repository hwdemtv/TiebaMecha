本手册由 **TiebaMecha** 核心开发团队根据多年贴吧矩阵运营经验沉淀而成。旨在帮助指挥官理解百度贴吧背后的审查模型，并有效利用机甲的防御装甲。

## 🧭 第一章：错误码解析 (Error Decoding)

在执行发帖、签到或回复任务时，如果日志中出现非零代码，请参考下表：

| 错误码 | 官方释义 | 深度解析与对策 | 机甲防御等级 |
| :--- | :--- | :--- | :---: |
| **0** | `SUCCESS` | 任务执行圆满成功。 | - |
| **404** | `Not Found` | **被秒删/已被删**。通常由于内容命中敏感词或账号权重极低。建议尝试“智能改写”。 | 🌕🌕🌕 |
| **205** | `Frequent` | **操作过于频繁**。由于短时间内请求过多导致 IP 或账号被限流。建议调高“拟人化延迟”。 | 🌕🌕🌕 |
| **-1** | `Refused` | **拒绝访问**。底层协议直接断开，通常是 IP 所在的网段被风控标记。建议启用高匿代理。 | 🌕🌕🌑 |
| **3150007** | `Auth Error` | **登录失效**。BDUSS/STOKEN 已过期。请在账号页面重新核验。 | - |
| **5** | `Locked` | **账号被封禁/禁言**。请检查账号是否还能正常回帖。 | - |

## 👻 第二章：幽灵帖 (Ghost Post) 深度诊断

### 什么是幽灵帖？
帖子发布成功（有 TID），作者可见，但**其他人看不见**（不出现在贴吧页，且登录其他号也搜索不到）。这是贴吧最隐蔽的风控机制。

### 为什么会产生？
1.  **关键词触发**：正文包含引流敏感词（如微信号、QQ 号、外部短链）。
2.  **账号权重过低**：新号、刚注册不久或长期未活跃账号。
3.  **设备环境关联**：多个账号使用了相同的设备指纹或 User-Agent。

### 核心诊断流程：
- [ ] **穿透测试**：在 Web UI 归档页点击 TID 链接。
- [ ] **隐身访问**：复制链接到浏览器“无痕模式”下打开。若提示“帖子已删除”，则是秒删；若页面显示但只有主楼，则是幽灵帖。
- [ ] **抗力对策**：开启 **AI 智能改写**，并确认已注入 **零宽混淆字符**。

## 🛡 第三章：机甲防御原理解析 (Cyber Shield)

TiebaMecha 部署了多层加固逻辑，以确保多号矩阵的生存：

### 1. BionicDelay™ (高斯演化延迟)
- **原理**：发帖间隔不再是死板的 `60s`，而是围绕均值呈 **高斯分布 (Normal Distribution)** 随机波动。
- **作用**：让后台扫描算法无法通过时间频率特征锁定机械行为。

### 2. 独立指纹隔离 (Device Isolation)
- **原理**：为每一个账号 ID 独立持久化一整套硬件指纹（CUID、TID、Model、IMEI 模拟）。
- **作用**：即使您在同一台电脑操作 50 个号，在贴吧后台看来也是 50 台分布在不同城市的独立手机终端。

### 3. Z-Width Injection (零宽混淆注入)
- **原理**：在正文的关键部分（如链接、数字）之间，静默插入人眼不可见的零宽字符 (`\u200b`, `\u200c`)。
- **作用**：通过改变文本的 Unicode 序列绕过基于正则匹配的自动删帖引擎。

## 📈 第四章：养号与权重维护 (Warming Up)

为了确保账号长盛不衰，建议遵循以下“养号”准则：

1.  **分批导入**：新号导入后，先静默 24 小时，只进行 `签到` 操作，不建议立即开启发帖。
2.  **模拟活跃**：在 [帖子管理] 模块，手动回复 1-2 条热门精品贴（非引流文案），建立良好的账号行为日志。
3.  **开启 BioWarming**：在 Web UI “账号列表”中点击“养号”开关。机甲将每 4 小时自动模拟一次完整的入吧阅读、点击与点赞行为。
4.  **代理对齐**：若条件允许，账号所在的代理 IP 与其绑定的省份贴吧尽量保持一致（逻辑地理一致性）。

### 💡 关于自动化养号的 Q&A
- **Q: 为什么定为 4 小时一次？**  
  A: 模拟真人一天“上线 3-4 次”的活跃规律，既能维持权重，又能避免过度请求导致 IP 封禁。
- **Q: 养号期间能同时发帖吗？**  
  A: 可以。养号逻辑由独立进程调度，与其发帖流程互不干扰，共同构建账号的信备等级。
