# 飞书知识库自动化创建计划 (Feishu Wiki Automation Plan)

## 1. 背景与目标
随着 **TiebaMecha** 项目功能的不断增加，为了提供更专业、更易于协作的文档体验，我们计划利用飞书（Lark）的云文档能力，构建一个动态更新、多端同步的项目知识库（Wiki）。

本计划的目标是实现从本地开发文档（Markdown）到飞书 Wiki 的自动化同步，降低维护成本，并提供一个美观的对外门户（Portal）。

## 2. 知识库架构设计

### 2.1 目录结构 (Hierarchy)
知识库将按照以下层级进行组织：

- **项目空间 (Projects)**
    - **贴吧机甲 (TiebaMecha)**
        - **用户指南 (User Guide)**: 同步自 `docs/USER_GUIDE.md`
        - **常见问题 (FAQ)**: 同步自 `docs/FAQ.md`
- **公告与动态 (News)**: 用于发布版本更新和项目动态。
- **贡献与开发者指南 (Development)**: 用于指导开发者参与贡献。

### 2.2 门户设计 (Portal Design)
首页（互为螺旋工具箱）将作为导航入口，包含：
- **项目愿景**：赛博机甲风格的百度贴吧自动化管理平台。
- **快速入口**：宫格形式的导航卡片，直达用户手册和 FAQ。
- **实时状态**：项目更新日志的预览。

## 3. 实现路线图

### 3.1 第一阶段：结构自动化 (Structure Setup)
- [x] 在飞书 Wiki 中建立根节点。
- [x] 自动化创建子页面及层级关系。
- [x] 完成首页 Portal 基础布局设计。

### 3.2 第二阶段：内容同步 (Content Sync)
- **同步逻辑**：读取本地 `docs/` 目录下的 Markdown 文件，解析内容并注入飞书编辑器。
- **格式优化**：针对飞书编辑器对 Markdown 的支持特性，优化图片嵌入（Image blocks）和代码块显示。
- **中文化适配**：确保所有页面标题及导航均使用简体中文。

### 3.3 第三阶段：持续化集成 (CI/CD)
- [ ] 开发 Python 同步脚本（基于 `lark-oapi`）。
- [ ] 集成到 Github Actions 或本地构建脚本中。
- [ ] 实现在本地文档更新并推送（Push）时，自动触发飞书 Wiki 的更新。

## 4. 技术规范与权限
- **权限依赖**：需要飞书应用（App）具备 `wiki:wiki` 和 `docx:document` 的读写权限。
- **API 选型**：优先使用 Feishu OpenAPI (v2)，兼容性强且支持区块化编辑。
- **安全保障**：所有 API 秘钥（App ID/Secret）需通过操作系统的环境变量或 `.env` 加密存储。
